# Week 2: Billing And External Pilot

## Objective
Move from pricing theory to a believable revenue path by defining a real billing decision and preparing for one external pilot outside PAPI.

## Why This Week Exists
- Pricing without billing or pilot readiness is still speculative.
- A single external pilot is worth more than broad internal optimism.
- We need a controlled first buyer path before talking about scale.

## Scope
1. Billing decision blueprint
2. External pilot onboarding flow
3. Sales shortlist and qualification logic
4. Pilot success criteria

## Deliverables
- Billing decision memo:
  - Stripe now
  - recurring invoicing first
  - hybrid/manual stopgap
- External pilot playbook
- Qualification checklist for first pilot prospects
- Success / failure scorecard for the first pilot

## Execution Tasks
### Billing
- Decide which option is realistic in the next 30 days:
  - Stripe Checkout
  - recurring invoicing
  - manual invoice + contract-first pilot
- Document what must be true before automated recurring billing is exposed.
- Define one canonical recurring revenue path; avoid parallel billing chaos.

### External Pilot
- Define what a first pilot includes:
  - setup
  - migration/manual configuration
  - support expectations
  - success metrics
- Define the minimum onboarding path for one external salon.
- Document what remains manual on purpose.

### Sales Shortlist
- Create qualification rules for the first 10-20 prospects.
- Prioritize salons where manual onboarding is still acceptable.
- Separate owner conversations from agency/partner conversations.

### Guardrails
- Do not expose automated billing if cancel/refund/support ownership is unclear.
- Do not start with more than one external pilot until the process is understandable.
- Do not let pilot promises outrun current onboarding reality.

## Acceptance Criteria
- One billing path is chosen as the canonical next step.
- A first external pilot can be explained end-to-end.
- Pilot qualification is simple enough to execute consistently.
- Revenue claims stay tied to real pilot/billing evidence.

## Review Checklist
- Is the first billing step realistic for the current product maturity?
- Can one external pilot be onboarded without product chaos?
- Is the support burden acknowledged honestly?
- Are pilot promises smaller than the actual product risk surface?
